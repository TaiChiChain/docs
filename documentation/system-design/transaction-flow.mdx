前面的章节我们从宏观上展示了 Axiomesh 区块链节点的网络拓扑结构、节点内部结构以及节点的组件组成关系。为了进一步了解 Axiomesh 区块链交易的全生命周期，我们沿着交易在验证者节点内部组件之间的流转路径，深入分析交易在每个组件
以及组件之间的处理逻辑。如下图所示，是一笔进入 Axiomesh 区块链节点的交易处理示意图：


![Axiomesh 区块链交易流程图](../../images/system-design/axiomesh-ledger-tx-flow-cn.png)

1. 提交交易

2. 验证交易

3. 交易进入内存池

4. 交易处理
    4.1 交易通过网络服务广播给其他节点
    4.2 交易通过共识服务打包（本节点是主节点的情况）

5. 共识后的交易列表传入执行器模块

6. 执行器执行交易，产生相应的数据集
    6.1 状态变更相关数据写入状态数据库
    6.2 非变更类数据，如区块数据、交易数据、交易回执数据写入 WAL 数据库
