---
title: '区块'
---

一个区块是一组交易的集合，包括链中上一个区块的哈希。这将区块链接在一起，形成一个链，因为哈希是通过对区块数据进行加密得出的。这可以防止欺诈，因为任何对以前任何一个区块的更改都会使后续所有区块无效，导致所有哈希都会改变，这将被运行区块链的所有参与者注意到。

# 为什么需要区块

为了确保 AxiomLedger 网络上的所有参与者保持同步状态并就交易的确切历史达成共识，我们将交易分成多个区块。这意味着同时有数十个（甚至数百个）交易被提交、达成一致并同步。

# 区块是怎样运作的

为了保留交易历史，区块严格排序（每个新创建的区块都包含对其父区块的引用），区块内的交易也严格排序。在任何特定时间，网络上的所有参与者都同意区块的确切数目和历史， 并且正在努力将当前的活动交易请求分批到下一个区块。

在随机选择的验证者在网络上构建完区块后，该区块将传播到整个网络；所有节点都将该区块添加至其区块链的末尾，然后挑选新的验证者来创建下一个区块。目前，确切的区块构建过程和提交/共识过程由 dBFT 协议规定。

# 一个区块包含了什么

一个区块包含大量信息。其中，区块头包括以下字段：

| 字段        | 简介                     |
| ----------- | ------------------------ |
| Number      | 区块高度                 |
| StateRoot   | 状态根哈希               |
| TxRoot      | 交易根哈希               |
| ReceiptRoot | 回执根哈希               |
| ParentHash  | 父区块哈希               |
| Timestamp   | 区块时间戳               |
| Version     | 区块链版本               |
| Bloom       | 包含了事件日志的数据结构 |
| GasPrice    | 区块的燃料费用           |

除此之外，区块还包含了该区块的所有交易和回执以及当前区块哈希。

# 区块大小

最后一个重点是，一个区块的大小是有界限的。每个区块最多可以包含 500 笔交易。每笔交易在区块中的燃料目标大小需要小于一亿。这非常重要，因为它确保了区块内交易的执行时间不会过长。如果区块中交易数量或者每笔交易的燃料限制可以无限扩大，由于空间和速度的限制，性能受限的全节点将逐渐无法跟上网络。区块越大，在下一个时间段内及时处理它们所需的计算能力就越强大。这将造成中心化，需要通过限制区块大小来解决这个问题。
