---
title: '区块'
---

一个区块是一组交易的集合。区块中包括了上一个区块的哈希，这将区块链接在一起，形成一条链。这种链式结构可以防止欺诈，因为哈希是通过对区块数据进行加密得出的，而对之前任何一个区块的更改都会使后续所有区块无效，导致所有哈希改变，这种变化将被运行区块链的所有参与者注意到。

## 为什么需要区块

为了确保 AxiomLedger 网络上的所有参与者保持同步状态并就交易的确切历史达成共识，我们将交易分成多个区块。这意味着同时有数十个（甚至数百个）交易被提交、达成一致并同步。

## 区块是怎样运作的

为了保留交易历史记录，区块采用了严格的排序方式，这意味着每个新创建的区块都包含对其前一个区块的引用，而区块内的交易也按顺序排列。在任何特定时刻，网络上的所有参与者都会就区块上的数据和交易历史记录达成一致，并努力将当前的活动交易请求打包成下一个区块。

一旦随机选择的验证者在网络上构建了一个完整的区块，该区块将传播到整个网络，其他节点都会将该区块添加到其区块链的末尾，然后选择新的验证者来创建下一个区块。目前，确切的区块构建过程以及提交和共识过程是由dBFT协议规定的。

## 一个区块包含了什么

一个区块包含大量信息。其中，区块头包括以下字段：

| 字段        | 简介                     |
| ----------- | ------------------------ |
| Number      | 区块高度                 |
| StateRoot   | 状态根哈希               |
| TxRoot      | 交易根哈希               |
| ReceiptRoot | 回执根哈希               |
| ParentHash  | 父区块哈希               |
| Timestamp   | 区块时间戳               |
| Version     | 区块链版本               |
| Bloom       | 包含了事件日志的数据结构 |
| GasPrice    | 区块的燃料费用           |

除此之外，区块还包含了该区块的所有交易和回执以及当前区块哈希。

## 区块大小

一个区块的大小是有限制的，每个区块最多可以包含 500 笔交易，而且每笔交易在区块中的燃料目标大小需要小于一亿。这种机制确保了区块内交易的执行时间不会过长。如果允许区块中交易数量或每笔交易的燃料限制无限扩大，由于空间和速度的限制，性能受限的全节点将逐渐无法跟上网络的发展。区块越大，就需要更强大的计算能力在下一个时隙及时处理它们，这将导致中心化问题。因此，有必要通过限制区块大小来解决这个问题。
