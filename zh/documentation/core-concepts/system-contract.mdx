---
title: "系统合约"
---

系统合约是AxiomLedger内置的合约，该合约无需部署，可以直接调用，系统合约的调用方式是和EVM保持兼容的。

系统合约使用特定地址，预留的地址段为0x1000-0xffff。目前系统合约主要包括以下类型：

| 合约地址     | 名称               | 描述                                      |
| ------------| -------------------|-------------------------------------------|
| 0x1000      | 治理提案ID生成合约  | 系统内部合约，用于生成治理提案的ID，保持递增               |
| 0x1001      | 治理合约          | 用于对AxiomLedger进行治理，主要包括提案和投票功能，提案的类型包括节点管理、委员会管理、白名单提供商管理、Gas管理等            |
| 0x1002      | AXC管理合约      | 该合约已被废弃 |
| 0x1003      | 地址映射名称合约    | 系统内部合约，用于将地址映射成唯一名称，便于标识地址        |
| 0x1004      | 白名单管理合约      | 用于管理可用于访问AxiomLedger的白名单，用作权限控制        |
| 0x1005      | 未完成提案管理合约    | 系统内部合约，用于管理未完成的提案        |
| 0x1006      | Epoch管理合约      | 用于管理链上epoch信息，支持查询等操作         |
| 0x1007      | AXC管理合约        | 管理AXC的份额、分配和转移        |
| 0x1008      | EntryPoint合约     | 智能账户相关合约，调用智能账户的入口，和以太坊抽象账户标准完全兼容        |
| 0x1009      | 智能账户工厂合约    | 负责生成智能账户，获取智能账户地址        |
| 0x1010      | Verifying Paymaster合约  | 负责验证智能账户的Paymaster部分签名，验证通过的账户可以免付Gas，由该Paymaster代付Gas        |
| 0x1011      | Token Paymaster合约   | 支持智能账户用其他类型Token支付Gas，Paymaster收取该Token并代为支付Gas        |
