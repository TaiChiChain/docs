---
title: "系统合约"
---

系统合约是AxiomLedger内置的合约，该合约无需部署，可以直接调用，系统合约的调用方式是和EVM保持兼容的。

系统合约使用特定地址，预留的地址段为0x1000-0xffff。目前系统合约主要包括以下类型：

| 合约地址     | 名称               | 描述                                      |
| ------------| -------------------|-------------------------------------------|
| 0x1000      | 治理提案ID生成合约  | 系统内部合约，用于生成治理提案的ID，保持递增               |
| 0x1001      | 节点管理合约        | 用于管理节点加入、移除、升级等功能            |
| 0x1002      | 委员会管理合约      | 用于选举治理委员会成员，治理委员会负责治理工作 |
| 0x1003      | 地址映射名称合约    | 系统内部合约，用于将地址映射成唯一名称，便于标识地址        |
| 0x1004      | 白名单管理合约      | 用于管理可用于访问AxiomLedger的白名单，用作权限控制        |
| 0x1005      | 白名单提供者管理合约 | 用于管理白名单提供者，只有白名单提供者可以调用白名单管理合约，对白名单进行更新        |
| 0x1006      | 未完成提案管理合约    | 系统内部合约，用于管理未完成的提案        |
| 0x1007      | Epoch管理合约      | 用于管理链上epoch信息，支持查询等操作         |
| 0x1008      | Gas管理合约        | 用于对gas相关参数进行调整，包括gas价格，gas价格上下限等        |
