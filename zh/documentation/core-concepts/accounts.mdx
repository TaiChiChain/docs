---
title: "账户"
---

AxiomLedger 使用账户模型。地址是账户的唯一标识符，生成账户需要私钥。一个账户具有许多属性，包括代币余额、投票权等。账户可以：

1. 发送交易以转移其代币，
2. 提出提案，
3. 为提案投票，
4. 部署智能合约，
5. 触发自身或他人发布的智能合约。
   AxiomLedger 上的所有活动都基于账户。

## 账户类型

AxiomLedger 拥有两种账户类型：

- 通用账户 - 也就是外部拥有账户（EOA），由拥有私钥的任何人控制。
- 合约账户 - 是部署在 AxiomLedger 网络上并由代码控制的智能合约。没有人拥有合约账户的私钥。

这两种账户类型都有以下能力：

- 接收、持有和发送 AXC 或代币
- 与已部署的智能合约进行交互。

## 账户与密钥对

一个账户由一个密码学密钥对组成：公钥和私钥。公钥可以映射到一个地址，而私钥用于签署交易。这个密钥对有助于证明交易实际上是由发送者签署的，并防止恶意攻击。

这可以防止恶意行为者广播虚假交易，因为链始终可以验证交易的发送者。

例如，如果 Alice 想从她的账户向 Bob 的账户发送 AXM，她必须创建并签署一个交易，然后将其发送到 AxiomLedger 网络进行验证。有了公钥，Alice 可以证明她最初发起了交易请求。如果没有加密机制，如果有个恶意攻击者 Cathy 可能能够公开广播类似的请求：“从 Alice 的账户发送 5 AXC 到 Cathy 的账户”，而没有人能够验证交易是否确实是由 Alice 本人请求的。

## 外部持有账户的创建

AxiomLedger 的密钥对生成算法与以太坊完全相同，使用椭圆曲线数字签名算法 ECDSA-secp256k1。密钥对生成的过程如下：

1. 首先，生成一个随机的私钥，由 64 个十六进制字符组成。
2. 然后，根据椭圆曲线数字签名算法从私钥生成公钥。
3. 最后，取公钥的 Keccak-256 结果的最后 20 个字节。

### 账户地址的格式

通过相同的私钥生成的 AxiomLedger 地址在十六进制格式上与以太坊地址完全相同。

```
0x8219e5E23396587Bf1969eFd1B211F379aa29494
```

## 合约账户

在智能合约部署到 AxiomLedger 网络时，会返回相应的合约账户地址，该地址是根据已部署的合约交易的 ID 和发送者的账户地址计算得出的。合约账户地址的格式与外部账户相同。例如

```
0x8219e5E23396587Bf1969eFd1B211F379aa29494
```
