本章将介绍 zkToken 协议。在整体架构方面，zkToken 利用 Pedersen 承诺的变种，其主要方案受到 PGC 的启发，尽管在细节上有些许差异。

## 孪生 Token 模型

zkToken 采用双令牌协议，确保了该协议与流行的公链项目的可组合性，同时支持明文账户和密文账户之间的灵活资金流动。
现有的隐私协议大多涉及对基础区块链的修改，这可以确保隐私，但往往会牺牲与其他协议的可组合性。
此外，许多隐私协议相对简单，将所有用户数据视为私密，没有区分明文和密文，更不用说它们之间的互动了。

DCPA 提出了孪生 Token 协议，以有效解决这两个问题。
用户通过孪生 Token 协议注册现有的 ERC20 代币，例如 USDT。
然后，该协议为这个 ERC20 代币部署一个相应的隐私合约，确保该合约中的数据是密文，并且不会影响原始的 ERC20 合约。

在实践中，如果用户选择明文交互，他们直接与原始的 ERC20 代币进行交互。
对于密文交互，他们与隐私 ERC20 代币进行交互。在涉及明文和密文之间的交互的情况下，例如涉及两种类型的交易时，
孪生 Token 路由合约将会辅助资金在这两个合约之间的转账。
