# v0.3.0
- 交易池支持价格优先的区块打包
- 优化系统合约和evm合约的互操作性
- 通过数据压缩减少p2p消息广播的数据量
- JSON-RPC接口支持trace和txpool
- 支持账本管理命令行
- 优化查询类接口的性能
- 支持基于状态快照的同步模式
- 重构节点秘钥管理
- 账本模块支持剪枝机制，可以在节点运行时对剪枝窗口外的账本历史状态进行裁剪，从而降低运行全节点所需的磁盘空间，并且提升账本读写性能
- 账本模块支持生成并校验指定区块的状态快照


# v0.2.0
第二个测试网Taurus版本，支持以下功能：
- 账本模块添加JMT树以及快照机制，支持查询和验证历史状态。
- 支持纪元机制，可通过纪元变更动态更新链的部分参数
- dBFT共识算法支持基于历史信誉的节点随机轮换和纪元变更时的验证者选举。
- 账本kv存储接入pebble db, 共识存储接入minfile以提升查询性能。
- 支持通过治理提案动态增删节点, 变更治理委员会成员，变更gas price参数, 节点版本升级。
- 支持非共识的候选者节点只同步数据。
- txpool和共识模块解耦，性能更高。
- 支持api层针对读写请求单独限流。
- p2p网络切换至gossip协议，对于大规模节点扩展性更好。
- 区块同步机制支持校验同步的区块是否正确。

# v0.1.0
初始发布版本，支持以下功能：
- EVM等效的API和SDK。
- 使用自研的dBFT共识算法，性能更高以及交易确认更快。
- 还支持内置的系统合约以方便链的管理。
- 支持和以太坊之间的可靠消息传输。
- 支持基于提案的链上链下结合的治理机制。
