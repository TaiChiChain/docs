---
title: "JSON-RPC介绍"
---

与Axiomesh交互需要向特定的JSON-RPC API方法发送请求。RPC请求必须发送到节点，响应必须使用某种传输协议返回到客户端。

## 介绍
Axiomesh的API使用名称空间系统。所有方法名称都由命名空间、下划线和命名空间中的实际方法名称组成。例如，eth_blockNum方法驻留在eth命名空间中。为了最大限度地与以太坊生态系统兼容，Axiom目前使用eth作为大多数接口的命名空间。

可以在每个命名空间的基础上启用对RPC方法的访问。

## 便利性库
虽然选择通过JSON-RPC API直接与Axiomesh接洽是可能的，但Dapp开发人员通常有更方便的途径。许多JavaScript和后端API库已经出现，在JSON-RPC API之上提供包装器。通过利用这些库，开发人员可以用他们喜欢的编程语言编写简单、简洁的方法，从而有效地支持JSON-RPC请求。这些请求与Axiomesh无缝对接，简化了开发过程。

## 兼容性
当前Axiom扩展了对大多数以太坊规范Jsonrpc接口的支持。然而，由于底层实现逻辑的差异，某些接口响应可能与以太坊的约定不同。如果出现任何兼容性问题，建议参考左侧边栏中特定接口的规范和使用示例以获得解决方案。
